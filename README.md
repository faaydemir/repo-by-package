## What is Repo by Package?

Repo by Package is a search engine that helps developers find GitHub repositories based on the packages and dependencies they use. Whether you're looking for React projects, Python applications using Django, or Rust projects with specific crates, this tool makes discovery simple and fast.

### Example Use Cases

- **Real-world React Applications**: Search for repositories using React to see real-world implementations and patterns
- **Discover Framework Combinations**: See how developers combine packages like Next.js + Tailwind CSS or Express + MongoDB
- **Learn from Examples**: Browse actual production code using specific libraries or dependencies
- **Research Best Practices**: Study how popular packages are implemented across different projects
- **Find Starter Templates**: Locate well-structured projects using your preferred tech stack

### Supported Languages & Ecosystems

- **[JavaScript/TypeScript](https://repobypackage.com/npm)**: npm packages (React, Vue, Next.js, Express, etc.)
- **[Python](https://repobypackage.com/pypi)**: pip packages (Django, Flask, FastAPI, pandas, etc.)
- **[Java/Kotlin](https://repobypackage.com/Maven)**: Maven dependencies
- **[Ruby](https://repobypackage.com/RubyGems)**: Gems
- **[Rust](https://repobypackage.com/cargo)**: Crates
- **[Go](https://repobypackage.com/go)**: Go modules
- **[C#/.NET](https://repobypackage.com/nuget)**: NuGet packages



## How It Works

1. **Continuous Crawling**: The application regularly crawls public GitHub repositories
2. **Dependency Extraction**: Parses package manifests (package.json, requirements.txt, go.mod, Cargo.toml, etc.)
3. **Indexed Search**: Creates a searchable database of repositories and their dependencies
